## Guidelines for Screenshots

1. **Resolution**: Use high-resolution images (at least 1080p for mobile, 1440p for web)
2. **Format**: PNG format preferred for crisp UI elements
3. **Content**: Use realistic but anonymized data
4. **Consistency**: Maintain consistent styling and branding across all screenshots
5. **Annotations**: Add callouts or highlights for key features where appropriate

## Adding New Screenshots

1. Take screenshots in the specified dimensions
2. Optimize images for web (compress without losing quality)
3. Update the README.md file if adding new screenshot categories
4. Ensure all screenshots follow the project's visual guidelines

## Demo Video

The demo video should be hosted on YouTube and embedded in the README.md. The video should cover:
- App overview and navigation
- Key features demonstration
- User workflows (health worker, villager, tourist)
- AI chatbot interaction
- Risk map functionality
- Alert system in action

Recommended video length: 3-5 minutes
